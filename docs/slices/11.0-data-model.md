# Slice 11.0 — Data-model expansion (design proposal)

Status: **PROPOSED — awaiting sign-off before any migration runs.**
Goal: add the rich fields the prototype's features need, **purely additively** (new
nullable columns + new tables; nothing dropped/renamed), then backfill existing
HVEDC data. Foundation for slices 11.1–11.12.

Design principles (per user directive: quality + maintainability):
- **Reuse before adding.** Where an existing table already models a concept, extend
  its vocabulary instead of adding a parallel store (avoids two sources of truth).
- **Real columns for flat, queryable, roll-up fields** (indexes, KPIs).
- **JSON columns only for genuinely nested / evolving structures** (matches the
  prototype's shapes; consistent with existing `settings`/`payload` Json usage).
- **New tables get the full tenant treatment**: `org_id` + RLS ENABLE/FORCE +
  `tenant_isolation` policy + composite `@@unique([id, orgId])` where junctions
  reference them + isolation-test coverage (cardinal rule #1).

**Team roles = reuse `project_links`.** The prototype's `team{architect, lender, GC…}`
maps to `project_links.role` (already company↔project with a role). We expand the role
vocabulary to the 20 disciplines; **open-role detection** = disciplines with no link.
This keeps ONE source of truth for "who's on the project" and inherits the composite-FK
cross-org guard for free. External (non-member) team members are out of scope for 11.0
(they can be added later as text, but every current HVEDC team member is a member org).

## 4. New tables (full tenant treatment)
- **Event** — `id, orgId, name, type, stage, date, venue, capacity, theme, description,
  notes, cost, projectId?`. Types: Member Dinner/Roundtable/Panel/Site Visit/Conference/
  Social/Other. Stages: Planning/Invitations Sent/Confirmed/Completed/Cancelled.
  `@@unique([id, orgId])` for the invitee composite FK.
- **EventInvitee** — `orgId, eventId, contactId?, externalName?, externalOrg?, rsvp, notes`.
  RSVP: Invited/Confirmed/Declined/Attended/No Show. Composite FKs `(eventId,orgId)` &
  `(contactId,orgId)` so an invitee can't straddle orgs (like meeting_attendees).
- **MembershipProposal** — `id, orgId, companyId, tier, amount, status, sentOn?,
  lastFollowUpAt?, driveUrl?, notes`. Status: draft/sent/negotiating/won/lost. Powers the
  dashboard "Proposals Sent" panel + revenue proposal pipeline (slice 11.11).

## 5. Backfill (re-reads item-8 source `Coterie_backup_2026-05-30.json`)
Additive UPDATEs, idempotent, run as owner inside the seed org's scope:
- **Restore Director/Advisory tier** — item-8 collapsed both `active` & `onboard` to
  status=`member` and dropped the distinction. Backfill sets `tier` from source `st`:
  `active`→Director Level, `onboard`→Advisory Level.
- Company: counties, dealSize, lookingFor, canOffer, agencyContacts, networkTags,

- Project: type, county, units, realizedValue, economicImpact, hvServices, fundingSources,
  stageHistory; expand existing project_links roles to the discipline vocab where source `team` maps.
